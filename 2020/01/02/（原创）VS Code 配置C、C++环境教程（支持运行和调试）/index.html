（原创）VS Code 配置C、C++环境教程（支持运行和调试）

发表于2020-01-02|更新于2021-07-04
|字数总计:1.9k|阅读时长:8分钟|阅读量:|评论数:

参考资料：https://blog.csdn.net/qq_43041976/article/details/100542557
    https://www.jb51.net/article/171027.htm
测试环境：Windows 7 旗舰版（x64）（虚拟机、已集成 2020/1/1 前所有补丁、已安装 VM Tools）、Windows 10 家庭中文版（x64）

第一步：下载安装 MinGW

下载 MinGW-W64 GCC-8.1.0 编译器（32 和 64 版本都有）
在线下载：MinGW-W64 GCC 下载地址
文件名为：MinGW-w64 - for 32 and 64 bit Windows
https://sourceforge.net/projects/mingw-w64/files/Toolchains%20targetting%20Win32/Personal%20Builds/mingw-builds/installer/mingw-w64-install.exe/download
下载完成之后，运行可执行文件，一直点击下一步即可（貌似需要科学上网）
离线下载：MinGW-W64 GCC 下载地址（推荐）
https://sourceforge.net/projects/mingw-w64/files/mingw-w64/mingw-w64-release/

注：文件名为 x86_64-8.1.0-release-win32-seh-rt_v6-rev0.7z
解压出来“mingw64”文件夹

第二步：环境配置

将解压出来的文件夹移入到：C:\Program Files (x86)，打开 mingw64 文件夹，里面就是编译器的所有文件
将文件夹加入到系统环境变量中（以 Windows 10 为例）：

C:\Program Files (x86)\mingw64\bin

Windows 7 为：
一定要在路径前面加上英文的分号
单击确定，重启电脑

第三步：下载安装 VSCode（已经装好了请跳到第四步）

  • vscode 请到官网下载最新版（推荐管理权限 64 位版本）
  • 下载地址：https://code.visualstudio.com/Download


等待下载完成，运行安装包：
注意在安装时一定要勾选上“添加到 PATH”选项！其他的任选

安装完成之后重启

第四步：安装 VS Code 相关的插件


然后 VS Code 会提示“restart now”，我们重启 VS Code
设置中文环境完成！

第五步：配置 C++的.vscode 文件

  • 在你写代码的地方新建文件夹和里面新建一个文件夹（文件夹名最好不要包含空格）

  • 然后在文件夹里面创建一个.vscode 文件夹

我的是：“C:\Users\Windows 7\Desktop\VSCode.code\C++”
在 Windows 7 下会出现“必须键入文件名的错误”，我们用 VS Code 打开文件夹“C++”
在 VS Code 中新建文件“1.cpp”，这时编译器会提示：
点击 install，等待安装完成。
为了规避 Windows 7 的重命名错误，我们这样做：

这时在 C:\Users\Windows 7\Desktop\VSCode.code\C++中会出现“.vscode”文件夹
然后在.vscode 文件夹里面创建三个文件，文件名如下
c_cpp_properties.json
launch.json（已有，无需创建）
tasks.json
请根据安装的位置来修改文件路径（如果跟我一样可以不用修改）

打开 launch.json，复制（覆盖之前的内容）：

launch.json 复制：

{
   “version”: “0.2.0”,
   “configurations”: [
       {
           “name”: “(gdb) Launch”, // 配置名称，将会在启动配置的下拉菜单中显示
           “type”: “cppdbg”, // 配置类型，这里只能为 cppdbg
           “request”: “launch”, // 请求配置类型，可以为 launch（启动）或 attach（附加）
           “program”: “${workspaceFolder}/${fileBasenameNoExtension}.exe”, // 将要进行调试的程序的路径
           “args”: [], // 程序调试时传递给程序的命令行参数，一般设为空即可
           “stopAtEntry”: false, // 设为 true 时程序将暂停在程序入口处，一般设置为 false
           “cwd”: “${workspaceFolder}”, // 调试程序时的工作目录，一般为${workspaceRoot}即代码所在目录 workspaceRoot 已被弃用，现改为 workspaceFolder
           “environment”: [],
           “externalConsole”: true, // 调试时是否显示控制台窗口，一般设置为 true 显示控制台
           “MIMode”: “gdb”,
           “miDebuggerPath”: “C:/Program Files (x86)/mingw64/bin/gdb.exe”, // miDebugger 的路径，注意这里要与 MinGw 的路径对应
           “preLaunchTask”: “g++”, // 调试会话开始前执行的任务，一般为编译程序，c++为 g++, c 为 gcc
           “setupCommands”: [
               {
                   “description”: “Enable pretty-printing for gdb”,
                   “text”: “-enable-pretty-printing”,
                   “ignoreFailures”: false
               }
           ]
       }
   ]
}

保存！

tasks.json，复制：

{
   “version”: “2.0.0”,
   “command”: “g++”,
   “args”: [
       “-g”,
       “${file}”,
       “-o”,
       “${fileBasenameNoExtension}.exe”
   ], // 编译命令参数
   “problemMatcher”: {
       “owner”: “cpp”,
       “fileLocation”: [
           “relative”,
           “${workspaceFolder}”
       ],
       “pattern”: {
           “regexp”: “^(.):(\d+):(\d+):\s+(warning|error):\s+(.)$”,
           “file”: 1,
           “line”: 2,
           “column”: 3,
           “severity”: 4,
           “message”: 5
       }
   }
}
